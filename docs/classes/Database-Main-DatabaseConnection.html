DatabaseConnection.php : 27

This class is used for establishing connections to databases and use them through actions performed by specialized objects and object collections. Methods for query execution and transactions are also available.

setCollectionTransactions()  : void
Enables or disables the automatic transactions for operations with collections. If enabled all operations that loops through a collection's items will happen inside a transaction, at the end the transaction will always be commited. If it is not desirable, disable these automatic transactions. Not using transactions can lower the application performance and cause inconsistencies due to the chance of selection of incomplete data. Although not all databases support transactions.
setPromiscuousMode()  : void
Enables or disables promiscuous mode. In this mode, methods that work with objects won't verify the object internal state before accepting it. For example, you will get an exception if you pass a newly created object for a delete method because since the object is marked as new, in theory it doesn't exists in the database yet. By enabling promiscuous mode a delete statement will always be executed, irregardless of the object state.
